SIP协议

SIP协议是由IETF标准化的应用层协议，用于不限于VoIP的通信。类似于HTTP协议，基于文本，具备请求响应结构，基于HTTP的Digest认证的用户认证机制。支持复合网络结构之间的通信，比如PSTN桥。能在UDP、TCP和SCTP协议上执行。

SIP协议的主要实体是终端（软手机或者物理手机），一个代理服务器，注册服务器，重定向服务器和一个位置服务器。注册、代理和重定向服务器可以合并也可以独立分开。终端和注册服务器通信说明其存在，这个信息存储在位置服务器中。一个用户可以同时被多台终端设备注册。执行CALL：终端和代理服务器通信，代理服务器使用位置服务器决定通话的路由位置，这个位置可以是同一个网络中的另外一部终端，也可以是另外一个网络中的另一个代理服务器。又或者，终端可以使用重定向服务器直接决定通话的指向位置。重定向服务器可以以同样的方式和位置服务器商议代理服务器在通话建立过程中的动作。一旦一个两个终端间端对端的信道建立起来，SIP通过SDP(Session Description Protocol)协议商议session参数(编码，RTP端口等)。

SIP协议的交换过程如图

SIP能够使用S/MIME(Secure Multipurpose Internet Mail Extensions) 携带复杂的认证载荷，包括公钥证书。当TCP被用于传输层协议时，TLS可以用于保护SIP消息。对于代理服务器，注册服务器和重定向服务器之间的通信，需要使用TLS。但只在终端和代理服务器或者注册服务器之间推荐使用TLS。IPsec也可以用于保护所有的通信，不仅仅是传输层。

VoIP威胁

Voice over IP Security Alliance(VoIPSA)指出：

  1. 社会工程学威胁：配置错误，bugs等。
  2. 偷听，拦截和修改威胁：攻击例子：路由重定向攻击和未加密的RTP的拦截。
  3. 拒绝服务攻击
  4. 滥用服务攻击
  5. 物理植入攻击
  6. 服务中断攻击

Security Analysis of Voice-over-IP Protocols(推荐)

文献链接

VOIP的协议栈如图1

本文的贡献是分析了VoIP协议栈中各层的安全性。特别的，我们集中于在不同的层之间互操作安全性，比如独立使用某个协议是安全的，而在不同层之间进行协议复合时可能就不是安全的。

  • 导致传输层的SRTP协议重复使用同一个加密密钥流进行加密，这样可以获得明文的异或结果甚至获得明文。SRTP的密钥流由AES产生，而AES的密钥由伪随机发生器PRF产生(session key)。而PRF的种子和session无关。
  • 针对ZRTP的密钥交换协议的攻击，迫使ZRTP的session参与者获取共享秘密。用于建立共享秘密的ZID值，不是ZRTP认证的一部分。这样，攻击者可以通过B（之前和A共享秘密）与A初始化session。作为session建立的一部分，A用于验证B是否知道共享的秘密。如果攻击者故意让这个验证失败，根据ZRTP规范，A将决定B已经忘记了这个共享秘密。
    简洁的说，ZRTP规范允许在共享秘密为空的情况下，继续执行协议。这样攻击者可以终止共享B与A之间的共享秘密。即使会话成员失去了共享秘密终止了协议，他们不能确认由语音计算的密钥并且必须终止通信。在这种情况下，攻击者完成了一次简单而有效的拒绝服务攻击。
